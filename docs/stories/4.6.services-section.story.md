# Story 4.6: Services Section with Cards

## Status
Draft

## Story
**As a** visitor,
**I want** to see a services/skills section with icon cards,
**so that** I understand what services are offered.

## Acceptance Criteria

1. Services section component created
2. Grid layout: 3 cards per row (desktop), 2 (tablet), 1 (mobile)
3. Each card has dark background (#1E1E1E), 1rem border radius
4. Cards have subtle drop shadow
5. Colored icon at top of each card (purple-pink gradient)
6. Service title and description included
7. Hover effect: slight elevation (scale 1.02 or translate)
8. Icons use Bootstrap Icons or similar icon set
9. Services data structure created in `data/services.ts`
10. Responsive grid adjusts properly on all breakpoints
11. Component tests verify card layout and hover states
12. Accessibility: proper heading hierarchy, alt text on icons

### ServiceCard Component Implementation
[Source: idea.md - Layout & grids section, changes.md section 6]

**File:** `src/components/home/ServiceCard.tsx`

```tsx
'use client';

import { Service } from '@/data/services';

interface ServiceCardProps {
  service: Service;
}

export default function ServiceCard({ service }: ServiceCardProps) {
  const Icon = service.icon;

  return (
    <div className="service-card group">
      {/* Icon */}
      <div className="mb-6 flex justify-center">
        <div className="icon-wrapper">
          <Icon className="icon" aria-hidden="true" />
        </div>
      </div>

      {/* Content */}
      <h3 className="mb-4 text-2xl font-bold text-white">{service.title}</h3>
      <p className="text-gray-300 leading-relaxed">{service.description}</p>
    </div>
  );
}
```

**CSS Styles (globals.css or component):**

```css
.service-card {
  background: #1e1e1e;
  border-radius: 1rem;
  padding: 3rem 2rem;
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.3);
  transition: all 0.3s ease;
  text-align: center;
}

.service-card:hover {
  transform: scale(1.02) translateY(-5px);
  box-shadow: 0 8px 12px rgba(0, 0, 0, 0.4);
}

.icon-wrapper {
  display: inline-flex;
  padding: 1.5rem;
  border-radius: 50%;
  background: linear-gradient(135deg, #450e93 0%, #d5007f 100%);
}

.icon {
  width: 3rem;
  height: 3rem;
  color: #ffffff;
}

/* Tailwind alternative */
.service-card-tw {
  @apply bg-[#1E1E1E] rounded-2xl p-12 shadow-lg transition-all duration-300;
  @apply hover:scale-[1.02] hover:-translate-y-2 hover:shadow-xl;
}
```

### Icon Gradient Implementation
[Source: idea.md - Icons & imagery section, Story 4.1]

**Gradient on Icons:**

Two approaches:

**Approach 1: Background Gradient (Recommended)**
```tsx
<div className="inline-flex p-6 rounded-full bg-gradient-to-br from-[#450E93] to-[#D5007F]">
  <Icon className="w-12 h-12 text-white" />
</div>
```

**Approach 2: Icon Color Gradient**
```tsx
<Icon
  className="w-12 h-12"
  style={{
    background: 'linear-gradient(135deg, #450E93 0%, #D5007F 100%)',
    WebkitBackgroundClip: 'text',
    WebkitTextFillColor: 'transparent',
  }}
/>
```

### Hover Animation Details
[Source: idea.md - Animations section]

**Transform Properties:**
- Scale: 1.02 (2% enlargement)
- TranslateY: -5px (5px lift)
- Transition: 0.3s ease

**Implementation:**
```css
.service-card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.service-card:hover {
  transform: scale(1.02) translateY(-5px);
}
```

### Component Testing
[Source: docs/architecture/testing-strategy.md]

**Test File:** `src/components/home/Services.test.tsx`

```typescript
import { render, screen } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import Services from './Services';
import { services } from '@/data/services';

describe('Services Section', () => {
  it('renders all services', () => {
    render(<Services />);

    services.forEach((service) => {
      expect(screen.getByText(service.title)).toBeInTheDocument();
      expect(screen.getByText(service.description)).toBeInTheDocument();
    });
  });

  it('renders section heading', () => {
    render(<Services />);
    expect(screen.getByRole('heading', { name: /what i do/i })).toBeInTheDocument();
  });

  it('renders grid layout', () => {
    render(<Services />);
    const grid = screen.getByRole('region', { name: /services/i }).querySelector('.services-grid');
    expect(grid).toBeInTheDocument();
  });

  it('has proper heading hierarchy', () => {
    render(<Services />);
    const h2 = screen.getByRole('heading', { level: 2 });
    const h3s = screen.getAllByRole('heading', { level: 3 });

    expect(h2).toBeInTheDocument();
    expect(h3s).toHaveLength(services.length);
  });
});
```

**ServiceCard Tests:**

import { render, screen } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import ServiceCard from './ServiceCard';
import { BsCameraVideo } from 'react-icons/bs';

const mockService = {
  id: 'test',
  title: 'Test Service',
  description: 'Test description',
  icon: BsCameraVideo,
};

describe('ServiceCard', () => {

    expect(icon).toBeInTheDocument();
  });
});

### Accessibility Requirements
[Source: idea.md - Semantic & accessible section]

**Semantic HTML:**
- Use `<section>` for Services section
- Use `<h2>` for section heading
- Use `<h3>` for card titles
- Use `<p>` for descriptions

**ARIA Attributes:**
- Icons: `aria-hidden="true"` (decorative)
- Section: `aria-labelledby` for heading
- Maintain heading hierarchy (h2 → h3)
