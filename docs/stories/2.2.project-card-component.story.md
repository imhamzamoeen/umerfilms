# Story 2.2: Project Card Component

## Status
Ready for Review

## Story
**As a** visitor,
**I want** to see project thumbnails that preview the content,
**so that** I can quickly scan and find projects that interest me.

## Acceptance Criteria

1. ProjectCard component displays thumbnail image, title, and category
2. Card has hover state with subtle animation (scale, overlay, or border effect)
3. Desktop: video preview plays on hover (first 3-5 seconds, muted)
4. Mobile: tap navigates directly to project (no hover preview)
5. Card is a clickable link to the project detail page
6. Card maintains consistent aspect ratio (16:9 recommended for video content)
7. Loading state shows placeholder/skeleton while thumbnail loads
8. Card is keyboard accessible (focusable, Enter to navigate)
9. Component accepts Project data as props

### Component Template Pattern
[Source: architecture/frontend-architecture.md#component-template]

```typescript
// src/components/portfolio/ProjectCard.tsx

import { forwardRef, useState } from 'react';
import Link from 'next/link';
import Image from 'next/image';
import { cn } from '@/lib/utils';
import { Project } from '@/types/project';
import { VideoPlayer } from '@/components/video';

interface ProjectCardProps {
  project: Project;
  priority?: boolean;
  showPreview?: boolean;
  className?: string;
}

export const ProjectCard = forwardRef<HTMLAnchorElement, ProjectCardProps>(
  ({ project, priority = false, showPreview = true, className }, ref) => {
    const [isHovered, setIsHovered] = useState(false);
    const [imageLoaded, setImageLoaded] = useState(false);

    return (
      <Link
        ref={ref}
        href={`/work/${project.slug}`}
        className={cn(
          'group block relative overflow-hidden rounded-lg',
          'transition-transform duration-300 hover:scale-105',
          'focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-amber-600 focus-visible:ring-offset-2',
          className
        )}
        onMouseEnter={() => setIsHovered(true)}
        onMouseLeave={() => setIsHovered(false)}
        aria-label={`${project.title} - ${project.category}`}
      >
        {/* Aspect ratio container */}
        <div className="aspect-video relative bg-gray-800">
          {/* Loading skeleton */}
          {!imageLoaded && (
            <div className="absolute inset-0 bg-gray-800 animate-pulse" />
          )}

          {/* Thumbnail image */}
          <Image
            src={project.thumbnailUrl}
            alt={project.title}
            fill
            className="object-cover"
            priority={priority}
            onLoadingComplete={() => setImageLoaded(true)}
          />

          {/* Hover video preview (desktop only) */}
          {showPreview && isHovered && (
            <div className="hidden md:block absolute inset-0">
              <VideoPlayer
                src={project.videoUrl}
                autoplay
                muted
                loop
                className="w-full h-full object-cover"
              />
            </div>
          )}

          {/* Hover overlay */}
          <div className="absolute inset-0 bg-black/20 opacity-0 group-hover:opacity-100 transition-opacity duration-300" />
        </div>

        {/* Card content */}
        <div className="p-4">
          <div className="flex items-start justify-between gap-2">
            <h3 className="text-lg md:text-xl font-semibold text-white group-hover:text-amber-600 transition-colors">
              {project.title}
            </h3>
            <span className="text-sm text-gray-400 whitespace-nowrap">
              {project.category}
            </span>
          </div>
        </div>
      </Link>
    );
  }
);

ProjectCard.displayName = 'ProjectCard';
```

### Hover Preview Implementation Details
[Source: architecture/components.md]

**Desktop (hover capability):**
- Detect with `@media (hover: hover)` or `hidden md:block` for video overlay
- On mouse enter: reveal VideoPlayer, autoplay muted
- On mouse leave: hide VideoPlayer, stop playback
- Video overlays thumbnail completely (absolute positioning)

**Mobile (no hover):**
- No video preview rendered (saves bandwidth)
- Tap activates Link immediately → navigates to project detail page
- Touch-friendly target size (entire card is clickable)

### Next.js Image Optimization
[Source: Next.js 15 docs, architecture/tech-stack.md]

**Image component props:**
- `src`: Thumbnail URL from project data
- `alt`: Project title for accessibility
- `fill`: Makes image fill parent container (responsive)
- `className="object-cover"`: Maintains aspect ratio, crops to fit
- `priority`: For above-fold images (first 3-4 cards)
- `onLoadingComplete`: Callback when image loads (hide skeleton)

**Lazy loading:**
- Next.js Image lazy loads by default (Intersection Observer)
- Only loads images when near viewport
- Use `priority` prop for hero/featured cards

### Accessibility Requirements
[Source: architecture/coding-standards.md, WCAG 2.1]

1. **Keyboard Navigation:**
   - Link is focusable (tabIndex 0 by default)
   - Focus visible ring (ring-2, ring-offset-2)
   - Enter key activates link

2. **Screen Readers:**
   - Image alt text describes project
   - aria-label on Link provides context
   - Category text is readable

3. **Color Contrast:**
   - Text on dark background (ensure sufficient contrast)
   - Hover state color change (amber accent)

### Responsive Design
[Source: architecture/frontend-architecture.md]

- Tablet (768px-1024px): 2 columns in grid
- Desktop (1024px+): 3 columns, hover preview enabled

**Spacing:**

### Hover Animation Options
[Source: AC 2 - subtle animation]

Choose ONE of these approaches:

1. **Scale Effect (Recommended):**
   ```css
   hover:scale-105 transition-transform duration-300
   ```

2. **Border Glow:**
   ```css
   hover:ring-2 hover:ring-amber-600 transition-all duration-300
   ```

3. **Overlay Effect:**
   ```css
   overlay: bg-black/20 opacity-0 group-hover:opacity-100
   ```

All three can be combined for a rich hover effect.
